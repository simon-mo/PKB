Sorting

  • Selection sort:

    • In iteration i, find the min, swap it with array[i]

    • Everything before it is sorted

    • Look for

      • Long length
      • Growing side of sorted values
      • Values being swapped

    • Best and Worst case:

    • Find the min will take each time

      • Slight optimization for finding is Heapsort find

      • Heapsort and Selection sort both take the smallest/largest unsorted element and insert it into the correct position

  • Insertion Sort:

    • Transverse from left to right

    • Insert the first item of unexamined portion into right piece of the sorted section on the left.

    • Look for:

      • Long length
      • Growing a sorted sublist
      • First element of unsorted sublist inserted into sorted sublist
      • Unsorted half unmodified except for first element

    • Worst case:

    • Best case:, when the list is already/almost sorted

    • Its runtime is based on the # of inversions

  • Heap Sort

    • Heapification first

      • Sinking based bottom up approach
      • Swimming based top down approach
      • Runtime of bottom-up heapification: O(n)
      • Runtime of top-down heapification: O(n log n)**

    • Then "pop" items

  • Merge Sort

    • Divide and Conquer

      • Divide into sublists of size 1
      • Repeatedly merge sublist to produce new sorted sublist
      • Can be top-down or bottom-up

    • Look for

      • Fewer iterations than sorts
      • Adjacent values being swapped during first pass
      • Multiple sorted sublists

    • Runtime: It's stable sort

  • Quick Sort

    • A pivot separate the arr to three section

      • A top-down process

    • Look for: Pivot and partitions on that pivot

    • Expected Runtime:

    • Worst case runtime, behave like insertion sort, a negatively sorted list/reverse sorted, runtime

    • Not STABLE

    • Advantage over Mergesort

      • Overall runtime is faster
      • Better caching/memory performance
      • When there are duplicates

    • Hoare Parititioning

    • The number of passes is the same as the number of digits

BST

  • Advantage:

    • Fast insertion/deletion/search
    • Self balancing

  • Form: B-trees and red-black trees

  • B-trees

    • (2,3) has max three children

    • (2,4) has max four children

    • Insertion procedure:

      • Add it to the bottom
      • if too much stuff in one node, move the 2nd element upward.
      • if the top has too much, split the tree, increase one level

  • LLRBs

    • One to one correspondence with 2,3 tree

    • Each two element node of (2,3) tree corresponds to a red link in an LLRB.

    • Invariant: Every path from root to leaf has the same number of black link

    • Rotation and recolor subtree

  • An easier way is:

    • Convert red black tree into a 2,3 tree
    • Do the insertion
    • And convert it back

Compression

  • Shannon-Fano Coding

    • Count relative frequencies of all characters in a text.
    • Split into ‘left’ and ‘right halves’ of roughly equal frequency.
    • Left half gets a leading zero.
    • Right half gets a leading one.
    • Repeat.

  • Huffman Encoding (More optimal)

    • calculate relative frequencies.
    • Assign each symbol to a node with weight = relative frequency.
    • Take the two smallest nodes and merge them into a super node with weight equal to sum of weights.
    • Repeat until everything is part of a tree.

  • Prefix Free Encoding

    • Looking for longest matching prefix
    • Translate it
